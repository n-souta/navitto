WordPress.org 申請の準備 - 資材メモ
====================================

1. readme.txt
   - ルートの readme.txt を整備済み。
   - Contributors に WordPress.org ユーザー名を記入すること。
   - Stable tag は実際に tags/ にあるバージョン（例: 1.0.1）と readme・プラグインヘッダーで一致させること。

5. SVN で WordPress.org に反映
   - 公式: https://developer.wordpress.org/plugins/wordpress-org/how-to-use-subversion/
   - リポジトリ URL: https://plugins.svn.wordpress.org/navitto/
   - 認証: WordPress.org プロフィールで「アプリケーションパスワード」を作成し、SVN のユーザー名（.org のユーザー名）とパスワード欄にそのパスワードを使う。
   - 手元から一括: プラグインルートで ./scripts/svn-wordpress-org.sh readme-assets
     （readme.txt と wordpress-org/ を trunk・assets・Stable tag フォルダに反映）
   - 本体コードのみ trunk 更新: ./scripts/svn-wordpress-org.sh trunk
   - 新リリース: trunk を更新したあと svn copy trunk tags/X.Y.Z -m "Tag X.Y.Z"（X.Y.Z は readme の Stable tag と揃える）
   - 作業用チェックアウトの場所は環境変数 WPORG_SVN_DIR で変更可（デフォルトはリポのひとつ上の navitto.svn-wporg）

2. スクリーンショット
   - 1枚以上（推奨4枚）、推奨サイズ 1280x720。
   - SVN の assets フォルダに screenshot-1.png, screenshot-2.png ...
   - 内容目安: (1)固定ナビ表示例 (2)投稿編集メタボックス (3)カスタマイザー (4)設定画面
   - 設定画面を写す場合はライセンス欄は写さない（キー漏洩防止）。範囲を切るかぼかすこと。

3. プラグインアイコン
   - 128x128 または 256x256（PNG/JPG）。
   - プラグインの assets/ に icon-128x128.png を配置。WordPress.org 表示用に SVN の assets にも同ファイルを配置。
   - 未作成の場合は 128x128 の PNG を用意（ナビ／目次を連想するシンプルな図案を推奨）。

4. ライセンス・GPL
   - navitto.php ヘッダー・readme.txt・LICENSE で GPL v2 表記済み。対応済み。
