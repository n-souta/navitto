# Navitto

**設定ゼロで始める固定ナビゲーション。** ナビっと表示、サクッと移動。

長文記事の目次から固定ナビを自動生成する WordPress プラグイン。テーマ内蔵目次・目次プラグインと連携し、H2 見出しを選択して表示できる。デザインプリセット（シンプル / テーマ準拠 / カスタム）とカスタマイザーで見た目を調整可能。

## 概要

Navitto は WordPress プラグインで、長文記事の読者体験を向上させます。投稿ごとに固定ナビの表示・非表示や表示する見出しを選べます。新規投稿のデフォルト有効化や、全投稿の一括有効・無効（設定は保持）も設定画面から行えます。

## 主な機能
- 目次自動連携（テーマ/プラグイン対応）
- H2見出し選択機能
- デザインプリセット3種類（シンプル・テーマ準拠・カスタム／カスタムは文字色・背景色・選択中テキスト色を指定可能）
- 横スクロール対応
- アイコン選択機能（見出しごとにアイコン設定可能）
- 新規投稿のデフォルト有効化オプション（設定 → Navitto）
- 一括適用（すべて有効 / すべて無効。設定を保持するため、再度切り替えで元に戻せる）

無料で利用可能。アイコン機能の有料版（¥1,000）を予定。

## 動作要件

- WordPress 6.0以上
- PHP 7.4以上

## 開発環境セットアップ

### 前提条件

- Git がインストールされていること
- Local by Flywheel（または同等のWordPressローカル環境）がインストールされていること
- GitHubアカウントへのアクセス権限があること

### 手順

#### 1. WordPressローカル環境を作成

1. Local by Flywheel を起動
2. 新しいサイトを作成（例: navitto-dev）
3. サイトの作成が完了するまで待機

#### 2. リポジトリをクローン

プラグインディレクトリに移動してクローンします。

- Windows: `cd "C:\Users\[ユーザー名]\Local Sites\[サイト名]\app\public\wp-content\plugins"`
- macOS: `cd ~/Local Sites/[サイト名]/app/public/wp-content/plugins`
- 実行: `git clone https://github.com/n-souta/navitto.git`

#### 3. ブランチの切り替えと最新取得

`cd navitto` のあと `git checkout main` および `git pull origin main`

#### 4. プラグインを有効化

管理画面 → プラグイン → インストール済みプラグイン → Navitto を有効化

### テスト用投稿の作成

動作確認には **H2見出し2つ以上** かつ **文字数3000文字以上** の投稿が必要です。投稿編集で複数の `## 見出し` と本文を入力してください。

### 動作確認

1. テスト投稿を作成し、投稿編集の「Navitto」メタボックスで表示モードを選択して公開
2. フロントで固定ナビが表示され、ナビ項目クリックで該当見出しにスクロールすることを確認

### 注意事項

- 開発中は `wp-config.php` の `WP_DEBUG` を `true` にすると便利です
- 本番環境では必ず `WP_DEBUG` を `false` にしてください
