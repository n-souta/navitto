# Navitto テストチェックリスト

テストフェーズで実施する項目の一覧です。

---

## 1. 基本機能テスト

- [ ] プラグイン有効化でエラーが出ない
- [ ] 投稿一覧・単一投稿でフロントが正常表示
- [ ] 条件を満たす投稿で固定ナビが表示される（H2数・文字数・メタボックス有効）
- [ ] 条件を満たさない投稿で固定ナビが表示されない
- [ ] スムーズスクロール・アクティブ項目ハイライトが動作する

---

## 2. 目次検出テスト

- [ ] テーマ内蔵目次（SWELL / JIN / SANGO 等）で連携する
- [ ] 目次プラグイン（TOC+ / Easy TOC 等）で連携する
- [ ] 検出優先順位どおりに目次ソースが選ばれる
- [ ] 目次がない記事では H2 から自前生成される

---

## 3. H2選択機能テスト

- [ ] 投稿編集で「表示する見出しを選択」が表示される
- [ ] 選択した H2 のみがナビに表示される
- [ ] カスタムテキストの編集・保存ができる
- [ ] 表示開始位置（即時 / N番目通過後 / px スクロール後）が動作する

---

## 4. デザインプリセットテスト

- [ ] カスタマイザーで「シンプル」「テーマ準拠」が選べる
- [ ] シンプルでデフォルトスタイルが適用される
- [ ] テーマ準拠でテーマの色が反映される
- [ ] 配置位置（上部固定 / 下部固定）が切り替わる
- [ ] ナビの高さ・フォントサイズが反映される

---

## 5. レスポンシブテスト

- [ ] 768px 前後で高さ・フォントが切り替わる
- [ ] 横スクロール・均等割りが期待どおり動作する
- [ ] 管理バー表示時も位置がずれない
- [ ] テーマの固定ヘッダーセレクタ（PC/SP）が効く

---

## 6. パフォーマンステスト

### 6-1. フロントエンド（読み込み・実行）

- [ ] **条件付き読み込み**: 固定ナビを表示しないページでは CSS/JS を読み込まない（is_singular('post') かつ should_display() で早期 return しているか）
- [ ] **スクリプト位置**: wp_enqueue_script( ..., true ) でフッター読み込みになっているか
- [ ] **依存**: フロント JS の依存が array('jquery') のみで過剰でないか
- [ ] **インラインCSS**: デフォルト値のみのときはインラインスタイルを出力しない（generate_inline_css の ':root {}' 判定）

### 6-2. クエリ・メタ

- [ ] **投稿メタ**: 表示判定・localize で get_post_meta が重複していないか（1投稿あたりの呼び出し回数）
- [ ] **テーマ mod**: get_theme_mod はキャッシュされるため許容。Detector 等で同一キーを何度も取っていないか
- [ ] **DB 直接クエリ**: 一括有効/無効（Settings）以外で $wpdb->query を使っていないか

### 6-3. アセット

- [ ] **CSS/JS サイズ**: frontend.css / frontend.js が極端に肥大化していないか
- [ ] **バージョン**: NAVITTO_VERSION でキャッシュバストしているか
- [ ] **管理画面**: 設定ページ・メタボックス用の CSS/JS は該当画面でのみ enqueue されているか

### 6-4. 確認コマンド・手順（任意）

- [ ] 固定ナビ非表示のページで「Network」に frontend.css / frontend.js が含まれないことを確認
- [ ] Query Monitor 等で単一投稿表示時のクエリ数・メタ取得回数を確認

---

## 7. セキュリティテスト

### 7-1. 投稿メタ保存（class-navitto-admin.php）

- [ ] **Nonce**: wp_nonce_field( 'navitto_save_meta', 'navitto_meta_nonce' ) を出力し、保存時に wp_verify_nonce( ..., 'navitto_save_meta' ) で検証しているか
- [ ] **権限**: 保存前に current_user_can( 'edit_post', $post_id ) でチェックしているか
- [ ] **サニタイズ**: $_POST['navitto_display_mode'] 等を sanitize_text_field( wp_unslash( ... ) ) しているか
- [ ] **ホワイトリスト**: display_mode / trigger_type / nav_width 等を許可値と照合しているか
- [ ] **URL**: カスタム項目の URL に esc_url_raw() を使用しているか

### 7-2. 設定ページ・一括操作（class-navitto-settings.php）

- [ ] **権限**: 設定表示・保存・一括操作で current_user_can( 'manage_options' ) をチェックしているか
- [ ] **AJAX nonce**: 一括操作で check_ajax_referer( 'navitto_bulk_action', 'nonce', false ) で検証しているか
- [ ] **SQL**: 一括更新で $wpdb->prepare() を使用し、プレースホルダで値をバインドしているか（LIKE は $wpdb->esc_like 使用）

### 7-3. 出力（エスケープ）

- [ ] **HTML**: 管理画面のラベル・説明文に esc_html_e() / esc_html() を使用しているか
- [ ] **属性**: value / placeholder / data-* に esc_attr() を使用しているか
- [ ] **URL**: リンクや href に esc_url() を使用しているか
- [ ] **localize**: wp_localize_script に渡す customTexts / customItems は保存時にサニタイズ済みか。JS 側で HTML に挿入する場合はエスケープしているか（XSS 防止）

### 7-4. カスタマイザー

- [ ] 各 setting に sanitize_callback が設定されているか（preset / position / number / checkbox / text）
- [ ] 許可値以外はサニタイズでデフォルトに落としているか（例: sanitize_preset）

### 7-5. アンインストール（uninstall.php）

- [ ] WP_UNINSTALL_PLUGIN 定義チェックで直接アクセスを防いでいるか
- [ ] メタ削除で $wpdb->prepare と $wpdb->esc_like を使用しているか

### 7-6. 禁止パターン（AGENTS.md）

- [ ] eval() / base64_decode による実行をしていないか
- [ ] プレフィックスなしのグローバル変数を使っていないか
- [ ] 未サニタイズの $_GET / $_POST を SQL や HTML に直接使っていないか
